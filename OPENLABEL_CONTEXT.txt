PRODUCT
-------
Name: OpenLabel
Type: Responsive Flutter app (Mobile, Web/Desktop layouts)
Elevator pitch: OSINT and Tech-Justice platform detecting hidden food anomalies and generating legal drafts.

TECH STACK
----------
- Flutter (Dart SDK ^3.11.1)
- flutter_riverpod — state; AsyncNotifierProvider
- go_router — declarative routes
- freezed + json_serializable — Data mapping
- dio — Live API client via Supabase Auth
- flutter_image_compress — image sizing (Mobile) + Base64 (Web)
- flutter_tts — Text-To-Speech integrations
- share_plus — Verdict exports
- image_picker — Cross-platform image selection

ROUTES (go_router)
------------------
Initial location: /home
  /home     — HomeScreen: Bento layout, scan card, Cart-Cop. (Web selects gallery natively avoiding bottom sheets)
  /loading  — LoadingScreen: Waiting state
  /verdict  — VerdictScreen: Hero Speedometer, Trust pill, TTS Playback, Flags, and FSSAI draft.

STATE & DATA FLOW
-----------------
AnalysisController (lib/providers/analysis_controller.dart):
  - analyzeImages(XFile front, XFile back) — compresses natively mapping Base64.
  - Returns result mapped dynamically into `translatedResultProvider` (translates live via translation_service 'hi', 'mr', 'en').

DATA MODELS (Freezed)
---------------------
ProductAnalysisResult (Snake-Cased API Binding):
  - scanId, productName, trustScore, trustLevel, overallVerdict, flags (List<FlagItem>)
  - fssaiNumber, legalDraftAvailable, legalDraftText, healthierAlternatives (List<String>)
  - allergyRisks, upfScore (double)
  - isNonEdible (bool default false)

DESIGN LANGUAGE (UI/UX)
-------------------------
- Dark mode default; OLED-style background #0A0A0A
- Surfaces/cards ~#1A1A1A, border #333333
- Accents: Neon Green #00FF66, Electric Red #FF3366, Warning Yellow #FFCC00
- Body text: grey-300 feel (#D1D5DB)
- Responsive: Handled by `Center > ConstrainedBox(maxWidth: 600/800)`
- Verdict top ~30% hero features an Animated Speedometer.
- Non-Edible items trigger a graceful UI intercept bypassing normal gauges showing an "Invalid Target" card.
- Text-To-Speech bindings feature strict manual play controls mapped dynamically to locale (`hi-IN`, etc.).
