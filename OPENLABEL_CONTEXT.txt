PRODUCT
-------
Name: OpenLabel
Type: Flutter mobile app (hackathon: 24h “OpenLabel” build)

Elevator pitch:
OpenLabel is a Consumer OSINT and Tech-Justice platform. Users either:
  (1) Scan physical food labels (front + back photos), or
  (2) Paste e-commerce product URLs (“Cart-Cop” for digital listings).

The app surfaces hidden sugars, questionable organic/cert claims, allergen
issues, and FSSAI-related concerns. When appropriate, it can draft a legal
consumer complaint (India-oriented scaffold; FSSAI mentioned in UI copy).

Current implementation: full UI + routing + integrated Live API via Dio & Supabase Auth.

TECH STACK
----------
- Flutter (Dart SDK ^3.11.1 in pubspec)
- flutter_riverpod — state; AnalysisController is AsyncNotifierProvider
- go_router — declarative routes
- freezed + json_serializable — FlagItem, ProductAnalysisResult
- dio — actively used for backend API client
- supabase_flutter — used for JWT extraction to authorize API requests
- flutter_image_compress — applied to encode local capture buffers for backend payload requirements
- google_fonts — Inter for UI; Merriweather for legal draft modal body
- image_picker — two camera captures for label scan (front then back)

ROUTES (go_router)
------------------
Initial location: /home

  /home     — HomeScreen: bento layout, scan card, Cart-Cop URL field
  /loading  — LoadingScreen: waits for AsyncNotifier analysis; on success → /verdict
  /verdict  — VerdictScreen: trust hero, flags, optional legal CTA + bottom sheet

Router is created in lib/router/app_router.dart as goRouterProvider (Provider<GoRouter>).
main.dart wraps the app in ProviderScope and MaterialApp.router with ref.watch(goRouterProvider).

STATE & DATA FLOW
-----------------
Provider: analysisControllerProvider
  Type: AsyncNotifierProvider<AnalysisController, ProductAnalysisResult?>

AnalysisController (lib/providers/analysis_controller.dart):
  - build() → null (idle / no result until run)
  - analyzeImages(File front, File back) — sets AsyncLoading, compresses images natively, posts base64 to /api/v1/scan/dual-image via ApiClient
  - analyzeUrl(String url) — sets AsyncLoading, posts to /api/v1/scan/link

Live result: Real result mapped from Dio client via `apiClientProvider`. Wait time is around 20-30s. Errors are caught as AsyncError to be bubbled up to UI.

User flow:
  1) Home triggers notifier (fire-and-forget), then context.push('/loading').
  2) Loading uses ref.listen; when state is AsyncData with non-null result → context.go('/verdict').
  3) Verdict reads ref.watch(analysisControllerProvider) and displays result.

DATA MODELS (Freezed)
---------------------
lib/models/product_analysis_result.dart

FlagItem:
  - code (String)
  - title (String)
  - severity (String)
  - evidence (String)
  - rationale (String)

HealthierAlternative:
  - name (String)
  - rationale (String)
  - link (String?)

ProductAnalysisResult (Snake-Cased Keys via @JsonKey):
  - scanId (String?)
  - productName (String?)
  - trustScore (double?)
  - trustLevel (String: RED / YELLOW / GREEN)
  - overallVerdict (String)
  - flags (List<FlagItem>)
  - fssaiNumber (String?)
  - legalDraftAvailable (bool)
  - legalDraftText (String?)
  - healthierAlternatives (List<HealthierAlternative>?)

Freezed classes are declared as `abstract class` (Freezed 3 expectation).

DESIGN LANGUAGE (UI/UX)
-------------------------
Goal: Modern premium startup UI — not default blocky Material.

- Dark mode default; OLED-style background #0A0A0A
- Surfaces/cards ~#1A1A1A, border #333333
- Accents: Neon Green #00FF66, Electric Red #FF3366, Warning Yellow #FFCC00
- Body text: grey-300 feel (#D1D5DB)
- Rounded corners 16–24px; bento-style grid on home
- Glassmorphism on verdict flag cards: BackdropFilter blur + translucent fill
- Home: large “Scan package” card with bounce scale animation on tap; Cart-Cop pill
  TextField with stronger neon border/glow when focused
- Verdict: top ~30% hero — features a custom hardware-accelerated Animated Speedometer projecting the trustScore;
  bottom gradient FAB-style “Draft FSSAI Complaint” when legalDraftAvailable
- Displays Healthier Alternatives gracefully if yielded by backend analysis.
- Legal content: near full-screen modal bottom sheet; serif (Merriweather); Copy button

Theme tokens live in lib/theme/theme.dart as OpenLabelTheme.dark.

PLATFORM NOTES
--------------
Android (android/app/src/main/AndroidManifest.xml):
  CAMERA, READ_MEDIA_IMAGES, READ_EXTERNAL_STORAGE (maxSdkVersion 32)

iOS (ios/Runner/Info.plist):
  NSCameraUsageDescription, NSPhotoLibraryUsageDescription

Scan uses ImageSource.camera twice (front then back). Simulators without camera
may need a device or a dev change to ImageSource.gallery.
